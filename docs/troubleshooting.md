# Troubleshooting Guide

Common issues and solutions for the Personal Brand Analytics platform.

## Installation Issues

### "Wrangler not found"

**Error:** `Error: Wrangler not found. Install with: npm install -g wrangler`

**Solution:**
```bash
# Install wrangler globally
npm install -g wrangler

# Or use npx
npx wrangler --version
```

### "GitHub token not found"

**Error:** `GitHub token not found. Set GITHUB_TOKEN environment variable.`

**Solution:**
```bash
# Authenticate with gh CLI
gh auth login --scopes repo,read:user,user:email

# Or set token manually
export GITHUB_TOKEN="your_token_here"
```

### "Google auth token not found"

**Error:** `Google auth token not found. Set up GOOGLE_APPLICATION_CREDENTIALS.`

**Solution:**
1. Go to [Google Cloud Console](https://console.cloud.google.com/)
2. Create service account
3. Download JSON key file
4. Set environment variable:
   ```bash
   export GOOGLE_APPLICATION_CREDENTIALS="/path/to/service-account.json"
   ```

## Repository Discovery Issues

### "API rate limit exceeded"

**Error:** `Error: API rate limit exceeded`

**Solutions:**
1. Use authenticated token (not anonymous)
2. Wait for rate limit to reset (usually 1 hour)
3. Use GitHub App token for higher limits
4. Enable caching in `.repo-cache` directory

### Repositories not classified correctly

**Symptom:** Core projects showing as PASSIVE_CLONE

**Solution:**
- Check `classification.require_pages` in config
- Add repos to `force_include_repos` list:
  ```json
  {
    "github": {
      "force_include_repos": ["my-important-project"]
    }
  }
  ```

## GA4/GTM Issues

### "Could not create GA4 property"

**Error:** `Warning: Could not create GA4 property: Permission denied`

**Solution:**
1. Verify service account has Analytics Admin permissions
2. Check OAuth scopes include `analytics.edit`
3. Or manually create property and set ID in config:
   ```json
   {
     "ga4": {
       "property_id_optional": "123456789",
       "measurement_id_optional": "G-XXXXXXXXXX"
     }
   }
   ```

### "Custom dimension already exists"

**Note:** This is informational, not an error. The script will skip creation if the dimension exists.

### GTM container not publishing

**Error:** `Warning: Could not publish container`

**Solution:**
1. Check GTM account has published version
2. Verify OAuth scopes include `tagmanager.manage.users`
3. Manually publish in GTM UI
4. Set container ID in config:
   ```json
   {
     "gtm": {
       "container_id_optional": "GTM-XXXXXXX"
     }
   }
   ```

## Repository Injection Issues

### "Could not push: Permission denied"

**Error:** `Could not push: remote: Permission denied`

**Solution:**
1. Verify you have write access to the repository
2. Check SSH keys or PAT has `repo` scope
3. Changes are committed locally - manually push:
   ```bash
   cd .repo-cache/repo-name
   git push -u origin add-analytics-tracking
   ```

### GTM snippet not appearing in HTML

**Symptom:** Script says success but snippet not found

**Possible causes:**
1. HTML file not in standard location
2. Missing `<head>` or `<body>` tags
3. File is generated by build process

**Solution:**
- Check supported locations:
  - `index.html` (root)
  - `docs/index.html`
  - `public/index.html`
  - `dist/index.html`
- For build-based sites, inject into source template
- Add custom path to config

### "analytics.config.json already exists"

**Note:** This means the repo was already processed. Check for drift:
- If config has correct `project_key`, no action needed
- If config needs update, delete and re-run

## Cloudflare Worker Issues

### "Worker deployment failed"

**Error:** `Error: Worker deployment failed`

**Solution:**
1. Check `CLOUDFLARE_API_TOKEN` is set
2. Verify account ID in `wrangler.toml`
3. Update wrangler:
   ```bash
   npm install -g wrangler@latest
   ```

### "Secrets not configured"

**Error:** `GA4 credentials not configured`

**Solution:**
```bash
# Set Worker secrets
wrangler secret put GA4_CREDENTIALS --name brand-analytics-proxy
wrangler secret put GA4_MEASUREMENT_ID --name brand-analytics-proxy

# Verify secrets
wrangler secret list --name brand-analytics-proxy
```

### Worker returns 500 error

**Symptom:** API calls fail with status 500

**Solution:**
1. Check Worker logs:
   ```bash
   wrangler tail brand-analytics-proxy
   ```
2. Verify GA4 credentials are valid JSON
3. Check measurement ID format (should be `G-XXXXXXXXXX`)

## Dashboard Issues

### "Failed to load dashboard data"

**Error:** Dashboard shows error message

**Solution:**
1. Check `PROXY_API_URL` is correctly configured
2. Verify Worker is deployed and responding:
   ```bash
   curl https://brand-analytics-proxy.<subdomain>.workers.dev/api/health
   ```
3. Check browser console for CORS errors
4. Verify `ALLOWED_ORIGINS` includes dashboard URL

### Charts not rendering

**Symptom:** Dashboard loads but charts are empty

**Solution:**
1. Check browser console for JavaScript errors
2. Verify Chart.js is loading:
   ```html
   <script src="https://cdn.jsdelivr.net/npm/chart.js@4.4.0/dist/chart.umd.min.js"></script>
   ```
3. Check API returns valid data
4. Clear browser cache

### "No projects tracked yet"

**Symptom:** Projects grid is empty

**Solution:**
1. Run `npm run apply` to process repositories
2. Check state.json has eligible repos
3. Verify repos have `strategy_applied: true`

## Playwright Test Issues

### "Test timeout exceeded"

**Error:** `Test timeout of 30000ms exceeded`

**Solution:**
```bash
# Increase timeout
export PLAYWRIGHT_TIMEOUT=60000
npx playwright test

# Or run headed to see what's happening
npx playwright test --headed
```

### "Browser not installed"

**Error:** `Browser executable not found`

**Solution:**
```bash
# Install Playwright browsers
npx playwright install

# Or install specific browser
npx playwright install chromium
```

## State Management Issues

### "state.json is corrupted"

**Symptom:** JSON parse errors

**Solution:**
```bash
# Restore from backup if exists
cp state/state.json.backup state/state.json

# Or reset state (will re-process all repos)
echo '{"last_updated": null, "repos": [], "summary": {...}}' > state/state.json
```

### Need to re-run for specific repo

**Solution:**
```bash
# Edit state.json and set strategy_applied: false for the repo
# Then run apply again
npm run apply
```

## Performance Issues

### "Script runs slowly"

**Symptom:** Discovery or apply takes too long

**Solution:**
1. Enable repo caching:
   ```bash
   export REPO_CACHE_DIR="./.repo-cache"
   ```
2. Use filters to process fewer repos:
   ```bash
   npm run apply -- --include-filter "important-"
   ```
3. Run in parallel (advanced - requires script modification)

## Debug Mode

Enable verbose logging:

```bash
export DEBUG=brand-analytics:*
npm run apply
```

Or edit `scripts/run.js` and add:
```javascript
process.env.DEBUG = 'brand-analytics:*';
```

## Getting Help

If issues persist:

1. Check logs in `reports/latest.json`
2. Review `state/state.json` for error messages
3. Enable debug mode
4. Check GitHub Issues for similar problems
